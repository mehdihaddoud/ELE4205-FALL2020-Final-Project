3 # Description du Projet
4 
5 Ce projet est une application client/serveur dans lequel le serveur
6  qui utlise l'Odroid-c2 pour capturer une image et envoie un unint32_t
7  au client via une connexion TCP/IP contenant un message:READY, IDOWN ou PUSHB.
8  Si le message envoyé est un READY cela signifie qu'il y a de la lumière et on peut transférer l'image.Le client demande alors l'image
9  et choisit la résolution dans laquelle il veut la recevoir en flux continu et peut changer de résolution à tout moment.
10  Si le message envoyé est un IDOWN cela signifie qu'il n'y a pas de lumière et pas d'image disponible,le client ne demande pas d'image.
11  Si le message envoyé est un PUSHB cela signifie qu'il y a de la lumière,que le boutton pression est enfoncé et une image est disponible,
12  le client fait un fork():le parent process affiche l'image comme dans le cas du READY tandis que le child process devra faire de la détection
13  de code QR et sauver l'image.Le client va se connecter sur un nouveau port(41000) pour envoyer
14  le nombre de caractère dans le code QR reçu suivi de la chaine de caractères.
15  Le nouveau serveur reçoit cette chaine de caratères et la joue en code morse ITU sur le buzzer.Après avoir terminé de jouer la
16  chaine de caractères,le programme en code morse se termine.

20 # Technologies utilisées
21  La capture de l'image et la détection de al luminosité se font du côté serveur
22  au travers d'un certains nombre d'équipements
23  - L'Odroid-c2
24  Il s'agit d'un ordinateur à carte unique développé par HardKernel et doté d'un processeur quad core 64 bits ARM Cortex-A53 , 2GHz.
25  il sert d'interface entre le serveur et le hardware du projet (camera, photorésistance, buzzer)
26  - Une caméra logitech (720p/30 ips) qui permet de capturer l'image en cas de luminosité
27  - un buzzer
28  - une photorésistance qui détecte le taux de luminosité
29  - un boutton poussoir dont l'état peut être lu par l'odroid via l'ADC et qui lorsqu'il est appuyé parmet au serveur
30  sur l'Odroid d'envoyer le message PUSHB au client sur la ligne de commande linux.
